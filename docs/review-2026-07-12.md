# Second-eyes review — 2026-07-12

Independent review pass run alongside the main working session (which owned the roadmap
queue). Method: 5 read-only internal auditors (skill prompts, bench methodology/statistics,
bot code, aggregation/calibration math, ops/doc drift) + 4 external researchers
(FutureSearch, Metaculus AIB ecosystem, academic literature, superforecasting/aggregation
theory), all candidates then individually re-verified by adversarial skeptics reading the
actual code, plus a completeness critic. 73 raw candidates → 16 confirmed, 3 plausible,
14 killed, 12 duplicates of roadmap items, 6 structural gaps. Everything below was
verified against source; line references are as of origin/main @ 64eb7e9.

The companion branch `fix/second-eyes-review` implements the mechanical fixes. Items
touching live production prompt text or requiring an A/B are NOT fixed here — they are
listed as decisions/experiments.

## Fixed in this branch

**Manifold bot (live mana — the journal now holds 2 real 25-mana bets, so these were live):**
- `eval_phase1` counted dry-run would-be bets in the promote/kill movement statistic
  (`if not bet: continue`, no `dry_run` check — unlike `open_exposure` and
  `_manage_phase2_positions`). The kill is permanent; ~50 aged paper bets could have
  triggered it (or a promotion) off bets never staked. Now excluded + regression tests.
- `already_bet_market_ids` treated any journaled bet — including dry-run paper bets from
  phase 0 or degraded phase-1 runs — as an open position, permanently blocking a later
  real bet on that market and starving the n≥50 promotion sample. Now excludes dry-run.
- `open_exposure` never decremented: nothing writes resolution status back to
  `bot/journal/manifold.jsonl`, so every placed bet counted as open exposure forever;
  once stale exposure crossed 30% of balance the bot would silently stop betting for
  good. Now takes an optional live-state lookup (reused from the phase-2 path, fail-closed
  on lookup errors, offline path unchanged).
- `place_bet` POST failure was journaled as "not placed"; a timeout-after-fill would
  therefore re-bet the same market once the 3-day dedupe expired (double stake). Failures
  are now journaled as `status: "unknown"` bets that keep the market guarded and count
  toward exposure; the response's bet id is verified before a bet counts as placed.
- A `--live` run that degrades to forecast-only (invalid key, balance-read failure,
  below the 1,100-mana floor) exited green forever while burning up to $5/hour.
  `run_manifold` now prints `BETTING-DISABLED: <reason>` on exactly the illegitimate
  paths (never phase-0/killed/no-divergence), and `manifold.yml` gains an alert step
  (one open issue, bot.yml's pattern) plus `issues: write`. The dead
  skip-missing-files loop in the commit step (a `continue` that filters nothing, making
  the leak-guard grep exit 2 on a missing journal) is replaced with a real
  existing-files filter with the same fail-closed semantics for present files.
- `score_manifold.py` docstring said the movement filter is 0.08; the code uses
  `DIVERGENCE_THRESHOLD` (0.05, amended 2026-07-11). Docstring now references the constant.

**Bench methodology:**
- `readout_tranche1.py`: the angles arm needs all its per-angle sub-runs to succeed to
  produce a row while plain/high need one, so angles has structurally higher, plausibly
  difficulty-correlated attrition — and pairing on set intersections then scores angles
  only on what it could complete (survivorship bias favoring the multi-run arm). The
  summary Brier/REL/RES table was also computed over each arm's own scorable set
  (non-comparable), and `--exclude-qid` was an unbounded manual lever on an n=40
  decision. The readout now prints per-arm run-0 coverage and missing qids, a
  three-way common-complete-case summary alongside the per-arm one (pairwise bootstraps
  unchanged — that math was correct), a differential-attrition diagnostic (high's Brier
  on angle-missing vs angle-complete questions; pre-registered verdict rule: coverage
  gap >5% AND ≥0.02 Brier difference ⇒ resume missing angles cells before applying the
  promote rule), and self-enforcing exclusions (every `--exclude-qid` must be a
  memory-screen run-0 candidate or the standard ECB exclusion, with printed provenance).
- `report.py` silently pooled all runs and dropped unknown tiers — run against the
  tranche file it produced a mislabeled, non-preregistered number. It now banners when
  fed unrecognized tiers or nonzero runs.
- `evidence_ablation.py` printed a run command using a `--blind` flag that does not
  exist (the command had never run), and its emitted set files carried the `resolution`
  field in plaintext readable by a zero-tier agent via Grep. Any prior
  evidence-elasticity result from this harness is invalid. Command fixed to
  `--leakfree none`, resolution stripped from emitted rows, structural guard added.
- `contamination_probe.py` computed the majority-class baseline over the full spec set
  but accuracy over the `--qids-from`/`--limit` subset. Display-honesty fix: baseline
  recomputed over the probed subset, both YES rates printed. The per-row contamination
  flags (what admissibility actually uses) never depended on this.
- `apply_recalibration` hardcoded the `[0.02, 0.98]` DEFAULTS clamp, so activating
  recalibration would silently tighten the bot's `[0.01, 0.99]` submission band.
  Dormant under the measured shrink fit (a=0.573 can't reach the tails) but wrong on
  activation with a stretch. The band is now a parameter; the bot passes its own.

**Docs:** HANDOVER's "0 bets so far" corrected (2 live bets, 50 mana exposure;
`score_manifold.py` is the authoritative count). CHANGELOG gains a retro-note that the
market_read bet-gate was removed by the 2026-07-11 policy amendment (the 0.4.17 entry
describes the old behavior).

## Not fixed here — decisions and experiments for the operator

1. **The portable skill's flagship instruction is the harness's measured null.**
   `aggregate.md`/SKILL.md's medium+ interactive default — research once, share the
   dossier, fan out k same-model reasoners — is exactly the shared-dossier pooling the
   bot measured at ~0.97 correlation and abandoned in v0.4.18 ("pool = member average at
   N× cost", run_bot.py's own comment). aggregate.md simultaneously asserts "separate
   contexts swing 2–3× wider / primary mechanism"; research-angles.md (independent
   research diets — the measured mechanism) is wired only into the harness and never
   referenced from SKILL.md. Needs: a doc reconciliation (which measurement is
   authoritative + a routing/precedence line in SKILL.md Step 4), and/or a 3-arm
   interactive A/B (shared-dossier fan-out vs independent-angle fan-out vs single run;
   kill the fan-out as a distinct tier if within |δ|<0.003 of a single run). Left alone
   because it changes shipped prompt text (operator-gated).
2. **Phase-1→2 promotion is contaminated by self-impact.** Selection deliberately
   includes thin books; a 25-mana bet on a ~25-bettor CPMM mechanically moves the price
   toward the forecast, so the toward-rate partly measures footprint. Preregister the
   correction before n approaches 50: score movement from the post-bet price at t+24h,
   or require the move to exceed the computable CPMM impact of the stake.
3. **Non-binary is an evaluation void** (every measured fact is binary-only; MC/numeric
   have no scoring path, no crowd baseline, no pooling rule) — build the ~40 MC + ~40
   numeric resolved pastcast slice with CRPS/multiclass scoring before trusting any
   non-binary submission. A read-only pinball-loss/log-loss track for the journal's
   continuous/MC book is the cheap first step.
4. **No live gate in the promote machinery** — every roadmap gate through step 8 is
   pastcast; step 7's scoreboard has no authority. Proposal: production-default status
   additionally requires non-negative paired delta vs the prior config on ≥30–40
   resolved prospective forecasts; treat pastcast-live sign disagreement as a leak alarm.
5. **KL-to-crowd distillation Goodharts against discrimination** (the measured gap is
   RES). Add a discrimination axis (Murphy RES, outcome-conditional spread) to
   report.py, tracked across versions, with a two-sided promote rule.
6. **The ~14 non-deadline tail catastrophes are undiagnosed** — the same post-mortem
   that found the deadline cluster, run over the rest of the tail (tag by qtype,
   horizon, direction, sources, crowd-miss), is the cheapest RES diagnostic available.
7. **Journal guards are not atomic cross-machine** (local `--live` + CI cron can
   double-position). The file-lock fix only covers local-vs-local; the real fix is a
   pre-bet position check against the Manifold API. Deferred as a design change.
8. **Corpus-on numbers are teacher-discovery-controlled**, not deployable-skill numbers
   (the manifest's membership is FutureSearch's reading list; slug text is searchable
   discovery metadata). Fine within-tranche (constant across arms); never quote a
   corpus-on Brier as the portable method's result. A ~30q corpus+vault vs vault-only
   pair sizes the discovery advantage if wanted.

## Verified-false (recorded so nobody re-litigates)

- geo_mean_odds does not extremize the shared-dossier pools (at ~0.03 disagreement all
  aggregators coincide; the "extremizes" warning attaches to drop_extremes, which is off).
- The FENCED_JSON regex handles nested payloads (backtracking anchored by the closing
  fence — tested by execution).
- `percentiles_to_cdf`'s min-step does not fail at 201+ outcomes (interpolation term
  dominates; actual failure threshold ~2,180 and unreachable in practice).
- `p_market or 0.0` cannot fire on eligible markets (CPMM prices are strictly interior;
  probability is a required schema field).
- The market_read/bet decoupling is the deliberate 2026-07-11 amendment (a preregistered
  hypothesis, journaled), not a missing guard; the capitulation readout is a pure query
  over already-journaled fields.
- The HANDOVER `sources:[]` suspect: build_record wiring is correct; not reproducible.
  The `n_searches = len(sources)` naming is the documented schema convention and does not
  double-count against the bench telemetry retrofit (independent producers/consumers).
- Hazard/window-scaling arithmetic as spine text is the measured prompt-ritual null
  (reasoning.md already draws that line explicitly); the deadline slice is owned by the
  roadmap's conditional research move instead.
