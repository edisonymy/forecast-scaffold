Pastcasting tool · adversarial validation

The concern about external validity was substantiated.

The tool passed basic archive retrieval after repair, but the audit found a live-redirect leak, live Wikipedia title-map dependencies, weak reliability under rate limits, legacy telemetry that credited attempts as evidence, and a binary-only outcome path. The paid legacy tranche therefore stays quarantined from decision-making.

Security and measurement defects repaired

Critical

Live redirect escape. A Wayback request could follow a redirect to the live origin while retaining the requested archive stamp. Effective responses must now match an anchored web.archive.org/web/<timestamp>id_/… replay—not a calendar, timemap, or toolbar page.

Critical

Wikipedia future leak. OpenSearch, redirects, and even an exact MediaWiki revision query consult today’s title-to-page map before date filtering. MediaWiki is no longer called: the exact title URL must have a cutoff-bounded Wayback capture or it is unavailable.

Critical

Undated corpus bypass. Agent calls could opt unparseable crawl dates into results. Undated rows now fail closed and the override is absent from the public tool schema.

Validity

Prospective rows skipped. frozen_at was not accepted as an as-of instant. It now is, with explicit precedence.

Validity

False research credit. Explanatory “unavailable” text counted as a read and source class. New content-free telemetry separates attempts, returned results, unavailable captures, and tool errors.

Reliability

Transient failures. Timeouts, 429s, and selected 5xx responses now retry with bounded backoff. The smoke still found persistent GDELT failure, so this is mitigation—not parity.

Decision contracts to freeze before spend

A cheap staircase, with stop rules.

18-fact external-validity gate · $0 model spend

Freeze facts spanning official pages, news, Wikipedia, and corpus discovery, with known pre-cutoff answers and load-bearing URLs.

Pass only if: 18/18 prevent post-cutoff leakage; ≥16/18 retrieve a correct pre-cutoff fact; ≥14/18 surface a load-bearing source in the top eight; zero unavailable captures are credited as reads. A security miss is automatic kill.

Research-parity pilot · maximum $12

Only after gate 1 passes: freeze nine unresolved, contamination-cleared binary questions and launch paired live-web and TimeVault arms within 30 minutes of the same frozen_at; TimeVault is pinned to that instant. Same Opus model, prompt, and concurrency 1. Run two pairs first and stop if projected total exceeds $12.

Pass only if: ≥8/9 valid pairs; zero leaks; median source recall ≥0.50 and TimeVault evidence precision ≥0.80. Two reviewers, blind to arm, label canonical URLs and factual claims; “load-bearing” means removing the claim could move the stated probability by ≥2 pp. Source recall is TimeVault/live intersection divided by live load-bearing URLs; precision is supported TimeVault load-bearing claims divided by all factual claims used in its rationale. Resolve reviewer disagreements while arm labels remain hidden. Single-draw forecast differences are descriptive only until same-tool repeatability is separately bounded.

Mechanism pilots, not global edits

For official-status/deadline retrieval, prospectively freeze 12 tagged questions plus four untouched controls, excluding the ten motivating holdouts. Score only resolved, paired rows; require ≥10 tagged pairs and all four controls or declare no-decision.

Promotion rule: mean paired Brier improvement Brier(current) − Brier(move) ≥0.015 on tagged rows and mean control degradation <0.003. Kill if control degradation ≥0.005; otherwise no-decision under this pilot. Numeric dispersion gets no promotion rule until the harness emits continuous distributions and CRPS/coverage/sharpness. No production prompt change is part of this PR.

Pull request scope

Measurement infrastructure, reproducible analysis, and this memo.

Code changes

  • Verify Wayback’s effective archive URL and remove every live Wikipedia title-map query.
  • Add bounded transient retries without treating persistent failure as success.
  • Recognize prospective frozen_at cutoffs and null crowd values.
  • Record semantic search/read success, unavailable captures, unique targets, and errors.
  • Correct report/source-class/readability handling for unavailable or null data.
  • Fail closed after a timeout; constrain positive benchmark budgets to serialized native Claude caps.

Reproducible diagnostics

  • minibench_2026_07_15.py reproduces binary disagreement and numeric width signatures.
  • pastcast_validity.py audits provenance and research mechanics without reading probabilities.
  • Regression tests cover redirect leakage, future title fallback, semantic telemetry, prospective rows, null crowds, and single-shot timeouts.
  • This HTML keeps claims, caveats, costs, nulls, and decision rules in one reviewable artifact.

Data provenance. The 15 MiniBench values were supplied by the operator from the closed-forecast UI on 15 July 2026. Prior numeric comparisons and BTF-2 diagnostics come from repository-frozen analysis artifacts. No updated question in this memo had resolved at analysis time. The journal contains 43 additional July 23–25 MiniBench-like forecasts, but they have neither community snapshots nor outcomes and are excluded from comparative claims.

Bottom line. Merge the safety and auditability repairs. Hold all forecasting behavior changes until the retrieval stack passes its external-validity gate and the proposed mechanism tests satisfy the frozen decision contracts above.
