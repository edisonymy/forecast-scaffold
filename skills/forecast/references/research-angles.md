# Research angles: engineered evidence diversity

Why this exists (2026-07): pooling same-model runs that share one research dossier is
worthless — measured: members disagree by only ~0.03 probability, so the pool equals the
member average while costing N times as much. Transcripts from the strongest public
forecasting ensemble show the alternative: members that research INDEPENDENTLY, with
assigned angles and deliberately different information diets (one member is market-blind
by design; search depths ranged 6 to 36 queries on the same question). Disagreement
born from different evidence is the only kind pooling can harvest.

Each angle below is a complete research-and-forecast brief. A harness assigns ONE angle
per independent run and pools the resulting forecasts; an interactive forecaster can run
them as separate passes (or parallel subagents) and pool by geometric mean of odds.
Every angle still owes the full output contract (probability, reasoning, sources,
reference class); the angle changes what you go looking for, never the honesty rules.

Before any angle: enumerate the 4–6 SOURCE CLASSES this question type needs (e.g. for
elections: rating agencies, poll aggregates, macro series, historical base rates,
special-election indicators, mechanical seat/threshold facts; for regulatory deadlines:
the official docket, procedural-step status, the institution's completion track record,
trade press). Your searches should cover classes, not just accumulate links.

## Angle F — fundamentals (market-blind by design)

Do NOT look up, cite, or anchor on any market price, community forecast, or aggregator —
even if other runs may; your value to the pool is an estimate UNANCHORED on the crowd.
Anchor on: the historical base rate for this event class (state the class and the count),
structural/macro drivers with official statistics, institutional ratings and datasets,
and how often situations like this one resolved YES over the relevant window. Status-quo
weighting: what happens if everyone stops acting? Set your number from the fundamentals
alone.

## Angle D — decomposition (bottom-up)

Break the resolution into its mechanical components: the sub-events, thresholds, seats,
procedural steps, or arithmetic that must ALL hold (or any-of hold) for YES. Estimate
each component from its own evidence — component-specific searches, not one broad query.
Recombine explicitly (state the combination rule: multiplication for conjunctions with
independence caveats, complements for disjunctions). Then sanity-check the recombined
number against the simple base rate: if they differ by more than 2x in odds, say which
you trust and why.

## Angle A — anomaly hunt (what would change the obvious answer?)

Start by stating the naive answer (base rate + status quo) in one line. Then spend the
run hunting for evidence AGAINST it: recent rating shifts, surprising polls or data
releases, procedural moves, filings, schedule changes, quiet reversals — the things that
make the naive answer stale. Recency-weighted: prefer the last 2–8 weeks. For each
anomaly found, price how much it actually moves the answer (many surprises are noise —
say so when they are). If the hunt finds nothing real, your forecast SHOULD sit near the
naive answer; finding nothing is a finding.
